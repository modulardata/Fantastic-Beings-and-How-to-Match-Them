Description

A samurai has no goal, only a path. This is not about us, as our game must have a goal â€” a condition for winning and losing :)

The goal of our game is to match a certain number of beings for a certain number of moves. For example, collect 10 Kelpie and 20 Zouwu in 15 moves. Let's get down to implementation!

Objectives
  • Draw the status bar first: add Moves and Score as shown in the Examples section. Leave some space next to them, draw the values of these fields, and give them IDs: moves-value and score-value. Use the Marmelad font from the Google Fonts collection;
  • Draw the left section of the status bar (bar ID - beings-for-win); place the icons and the number of creatures you need to match to win there. The number of creatures should be placed in the span tag with the appropriate class, for example:
<span class="zouwu">3</span>
<span class="kelpie">0</span>
  • To win the game, you need to collect 3 Zouwu and 0 Kelpies in one move. These are the simplified conditions required for testing; you may set other conditions outside this stage. Calculate points as follows: 10 points for each matched creature;
  • Create a new element at the bottom of the page that displays the game result (with the game-footer id and this background image). Use the Marmelad font from the Google Fonts collection. The initial value for this item is the game instruction:
Swap animals to form a sequence of three in a row
  • Change the contents of the new element with a message about victory if the player managed to collect all the required creatures in a given number of moves or less:
You won! Reload the page to start the game again.
  • Output the following message about the loss if there're no moves left, but the required number of creatures is yet to be reached:
You lost! Reload the page to start the game again.
  • After winning or losing the game, a user should not be able to continue the game and select creatures on the map.
